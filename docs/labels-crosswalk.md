# Runtime Block Labels Crosswalk

This document maps runtime block types to their display labels and the logic used to generate them. It serves as a reference for maintaining consistency across the UI (Review Grid, Clock, and History).

## 1. Compiled Blocks (via `LabelComposer`)

Most blocks are generated from parsed script statements. Their labels follow a standardized pattern:
**`[Logic] [Metric] [Identity] [Attributes]`**

| Block Type | Default Label Fallback | Label Composition Logic | Proposed Changes / Notes |
| :--- | :--- | :--- | :--- |
| **Effort** | `Effort` | `LabelComposer.build(statements)` | |
| **AMRAP** | `AMRAP {X} min` | `LabelComposer.build(statements)` | |
| **EMOM** | `EMOM {X}` | `LabelComposer.build(statements)` | |
| **Rounds** | `{X} Rounds` or `21-15-9` | `LabelComposer.build(statements)` | |
| **Timer** | `Timer` | `LabelComposer.build(statements)` | |
| **Group** | `Group` | `LabelComposer.build(statements)` | |

### `LabelComposer` Component Logic

| Component | Source | Logic / Priority |
| :--- | :--- | :--- |
| **Explicit Label** | `Label` fragment | If a `Label` fragment exists in the script, it overrides everything else. |
| **Logic** | Hints or Fragments | Priority: `AMRAP`, `EMOM`, `TABATA`, `FOR TIME`. Derived from dialect hints or specific keywords. |
| **Metric** | `Rounds` or `Duration` | Priority: Rounds sequence (e.g., `21-15-9`), Rounds count (e.g., `10 Rounds`), then Duration (e.g., `5:00`). |
| **Identity** | Other Fragments | Joins non-metric, non-logic fragments (e.g., `100 KB Swings`). Includes interleaved reps. |
| **Attributes** | `Resistance` / `Distance` | Appends weight or distance targets (e.g., `95lb`, `400m`). |

## 2. System & Special Blocks

These blocks are often auto-generated by the runtime or compiler and have fixed or simple dynamic labels.

| Block Type | Current Label | Generation Logic | Proposed Changes / Notes |
| :--- | :--- | :--- | :--- |
| **SessionRoot** | `Session` | Defaults to `config.label ?? 'Session'`. Usually "Morning WOD" or similar from script metadata. | |
| **WaitingToStart**| `Ready to Start` | Hardcoded in `WaitingToStartBlock` constructor. | |
| **Rest** | `Rest` | Defaults to `config.label ?? 'Rest'`. Auto-generated between intervals/rounds. | |
| **Idle** | `Idle` | Used when the runtime is active but no workout is loaded. | |

## 3. Label Consistency Rules

*   **Case Sensitivity**: Logic keywords (AMRAP, EMOM) should be UPPERCASE. Identity text (exercise names) should follow the script's casing (usually Sentence Case).
*   **Metric Formatting**:
    *   Durations: `MM:SS` or `H:MM:SS`. Short durations under a minute can be `:SS`.
    *   Rounds: Use `Rounds` suffix for counts (e.g., `10 Rounds`), but no suffix for sequences (e.g., `21-15-9`).
*   **Spacing**: Single space between components. `LabelComposer` handles collapsing redundant whitespace.
*   **Fallback**: If no fragments are available and no default is provided, the label defaults to `Block`.

## 4. Proposed Improvements

| Area | Current Behavior | Desired Behavior |
| :--- | :--- | :--- |
| **Empty Blocks** | Defaults to `Block` | Should probably show the first line of text or "Untitled Block". |
| **Nested Groups** | Often just says `Group` | Should attempt to summarize children (e.g., "3 Exercises") if no label is provided. |
| **Rest Labels** | Just says `Rest` | Could include what's coming next (e.g., `Rest (Next: Swings)`). |
