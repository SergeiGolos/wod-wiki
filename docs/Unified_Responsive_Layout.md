## Core Concept: The Panel Strip

The entire application is a **6-panel horizontal strip** that the viewport slides across:

| Position | Panel | Width | View |
|----------|-------|-------|------|
| 1 | **Monaco Editor** | 2/3 | Plan |
| 2 | **EditorIndexPanel** | 1/3 | Plan |
| 3 | **TimerIndexPanel** | 1/3 | Track |
| 4 | **TimerDisplay** | 2/3 | Track |
| 5 | **AnalyticsIndexPanel** | 1/3 | Analyze |
| 6 | **TimelineView** | 2/3 | Analyze |

### Panel Naming Convention

All 1/3-width panels follow the `*IndexPanel` naming pattern:
- **EditorIndexPanel** - Document structure browser (Plan view)
- **TimerIndexPanel** - Live execution history (Track view)
- **AnalyticsIndexPanel** - Segment tree for analysis (Analyze view)

### Viewport Positions

The viewport shows **exactly 3/3 (100% width)** at any time, revealing different panel combinations:

```
PLAN VIEW:      [Editor 2/3][EditorIndex 1/3]
                 â–² Panels 1-2

TRACK VIEW:     [TimerIndex 1/3][Timer 2/3]
                 â–² Panels 3-4

ANALYZE VIEW:   [AnalyticsIndex 1/3][Timeline 2/3]
                 â–² Panels 5-6
```

## View Configurations

### PLAN View (Panels 1-2)

**Purpose:** Edit and browse workouts

| Slot | Panel | Content |
|------|-------|---------|
| Left (2/3) | **Monaco Editor** | Markdown + WOD blocks |
| Right (1/3) | **EditorIndexPanel** | Document structure with expandable WOD details |

**User Flow:**
1. Edit workout in Monaco Editor
2. Browse document structure in EditorIndexPanel
3. Click WOD block â†’ expands in-place to show context + "Start Workout" button
4. Click "Start Workout" â†’ viewport slides to TRACK view

### TRACK View (Panels 3-4)

**Purpose:** Execute workout in real-time

| Slot | Panel | Content |
|------|-------|---------|
| Left (1/3) | **TimerIndexPanel** | Live execution tree + active context |
| Center (2/3) | **TimerDisplay** | Large timer + controls |
| Right (1/3)* | **RuntimeDebugPanel** | Stack + memory (debug mode only) |

*Debug panel is an overlay, not part of the strip

**User Flow:**
1. Timer runs automatically or on user command
2. History builds in real-time (newest at bottom)
3. Click "Stop" â†’ viewport slides to ANALYZE view

### ANALYZE View (Panels 5-6)

**Purpose:** Review completed workout data

| Slot | Panel | Content |
|------|-------|---------|
| Left (1/3) | **AnalyticsIndexPanel** | Segment tree (selectable) |
| Right (2/3) | **TimelineView** | Charts + data tables |

**User Flow:**
1. Select segments in left tree
2. View filtered analytics in right panel
3. Click "Plan" â†’ viewport slides back to PLAN view

## Panel Definitions

### Panel 1: Monaco Editor (2/3)

```typescript
interface EditorPanel {
  width: '2/3';
  visibleIn: ['plan'];
  content: MonacoEditor;
  interactions: {
    onContentChange: (content: string) => void;
    onBlockSelect: (blockId: string) => void;
  };
}
```

### Panel 2: EditorIndexPanel (1/3)

**Key Change:** Merge `WodIndexPanel` and `WorkoutContextPanel` into a single expandable panel.

```typescript
interface EditorIndexPanel {
  width: '1/3';
  visibleIn: ['plan'];
  content: {
    documentItems: DocumentItem[];
    expandedBlockId: string | null;  // Which block is expanded
  };
  interactions: {
    onBlockClick: (blockId: string) => void;      // Expands/collapses
    onStartWorkout: (blockId: string) => void;    // Slides to TRACK
  };
}
```

**Expand/Collapse Behavior:**
- Collapsed: Shows summary (title, parsed status badge)
- Expanded: Shows full statement list + "Start Workout" button
- Only one block can be expanded at a time

### Panel 3: TimerIndexPanel (1/3)

```typescript
interface TimerIndexPanel {
  width: '1/3';
  visibleIn: ['track'];
  content: {
    activeContext: WodBlock;           // Currently running workout
    executionHistory: Segment[];       // Completed segments
    activeSegmentIds: Set<number>;     // Currently on stack
  };
}
```

### Panel 4: TimerDisplay (2/3)

```typescript
interface TimerPanel {
  width: '2/3';
  visibleIn: ['track'];
  content: {
    elapsedTime: number;
    status: 'idle' | 'running' | 'paused';
  };
  interactions: {
    onStart: () => void;
    onPause: () => void;
    onStop: () => void;      // Slides to ANALYZE
    onNext: () => void;      // Skip to next segment
  };
}
```

### Panel 5: AnalyticsIndexPanel (1/3)

```typescript
interface AnalyticsIndexPanel {
  width: '1/3';
  visibleIn: ['analyze'];
  content: {
    segments: Segment[];
    selectedIds: Set<number>;
  };
  interactions: {
    onSelectSegment: (id: number) => void;  // Toggle selection
  };
}
```

### Panel 6: TimelineView (2/3)

```typescript
interface TimelinePanel {
  width: '2/3';
  visibleIn: ['analyze'];
  content: {
    rawData: TelemetryPoint[];
    segments: Segment[];
    selectedSegmentIds: Set<number>;
  };
}

**Click Behavior:**
```typescript
const handleBlockClick = (item: DocumentItem) => {
  if (item.type !== 'wod') return;
  
  // Toggle expansion
  if (expandedBlockId === item.id) {
    setExpandedBlockId(null);  // Collapse
  } else {
    setExpandedBlockId(item.id);  // Expand (auto-collapses previous)
  }
};

```typescript

## Summary

The sliding viewport model provides:

1. **Mental clarity** - Users understand they're moving through a linear workflow
2. **Smooth transitions** - CSS transforms for performant sliding
3. **Unified codebase** - One component for desktop and mobile
4. **Simplified state** - View position + panel-specific state
5. **Natural gestures** - Swipe on mobile maps to slide direction

Key changes from current implementation:
- **Merge** `WodIndexPanel` + `WorkoutContextPanel` â†’ expand/collapse in-place
- **Replace** view mode switching with horizontal slide transitions
- **Add** debug mode as 1/3-1/3-1/3 layout option in Track view
