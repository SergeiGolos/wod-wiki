The runnable Markdown.

The pitch: a vertical Run rail lives right of the editor. Each button is a real useWodBlockCommands command — same set the per-block context menu exposes today. The result row is the live output, painted with the same cdlColumnDefinitions the review grid uses.

The same Markdown, two surfaces: an editor on the left that you can edit freely, a Run rail on the right that you can act on. The rail is the entire Phase 2 — there is no separate "tracker screen" in the hero, the buttons are the affordance.

Below the editor sits a one-row ResultsView preview. The Rep cell lights up when you hover a wod line — a small affordance that says "this is the line that produced that number".

All three — note, run rail, result — share the same Rep cell. Edit the number, the row's Rep column updates on the next keystroke.
