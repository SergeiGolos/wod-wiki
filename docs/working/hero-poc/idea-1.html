The whiteboard script

Plan in plain text. Run in one click. Read in a row.

One Markdown file, one workout, one row in the journal — and one metric, Rep, that travels from your plan to the runtime to the result.

1 · Plan

Write the workout in free-form Markdown.

The wod block is just fenced Markdown. Edit the line, change 10 Pushups to 12, and the Rep metric is re-emitted on the next keystroke.

2 · Run

Run, schedule, or share the block.

The same useWodBlockCommands hooks the editor uses today — Run this block, Add to today, Schedule, Share — sit right beside the wod fence.

3 · Read

The result lands in the same table shape.

A ResultsView row with the columns you already use: Date · Effort · Reps · Volume · MET-min · TIS. The Rep cell is the same metric, with a new user-origin value.

Anchor metric: Rep. Parser plans it. Runtime reports it. User confirms it. Analytics re-derives it.
