One metric, three places

One Rep. Three places it lives.

The pitch: type 10 Pushups in the note, run the block, the same metric moves through the parser, the runtime, and the result row — a different origin stamp each time.

A line of Markdown like 10 Pushups produces a single Rep metric. The metric travels through the same Markdown file, picking up a new origin at every stage. The right panel shows the same number four times — once per origin tier — with the value the tier actually owns.

The big number is the visible value: the MetricOwnershipLedger selects the lowest-tier origin that has a value. Tiers (highest first): parser, dialect, compiler, runtime, user. So your real reps (user, tier 0) replace the planned reps (parser, tier 3) the moment you log them.

The single number on the right is the source of truth. The three cards underneath show the history that produced it.
