# AudioToggle

The mute/unmute button for the in-app audio system; it reads and writes audio state through `AudioContext`, so no props are required.

## Depends On

| System / Token | Usage |
|----------------|-------|
| `AudioContext` / `useAudio()` | Provides `isEnabled` state and `toggleAudio` callback |
| `AudioProvider` | Supplied globally by `StorybookHost` — no extra setup needed in stories |
| Lucide icons | `Volume2` (enabled) / `VolumeX` (muted) |

## Notifies

| Callback / Event | Trigger | Payload |
|-----------------|---------|---------|
| `toggleAudio()` | User clicks the button | none |

## Data Shapes

### Props

None — fully context-driven.

### Internal State

| State | Type | Initial | Purpose |
|-------|------|---------|---------|
| `isEnabled` (context) | `boolean` | `true` | Whether audio is on; sourced from `AudioContext` |
