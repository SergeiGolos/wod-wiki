# Tracker — Mobile

The mobile-viewport layout for the workout tracker; stacks the timer controls above the visual state panel for narrow screens, using the same real `ScriptRuntime` as the web variant.

## Depends On

| System / Token | Usage |
|----------------|-------|
| `TimerScreen` | Timer controls panel |
| `VisualStatePanel` | Runtime stack visual state |
| `ScriptRuntime` | Real runtime (no mocks) |
| Viewport: 375×812 px | Portrait mobile simulation |

## Notifies

| Callback / Event | Trigger | Payload |
|-----------------|---------|---------|
| Start / Pause / Stop / Next | Timer control buttons | none (dispatched to runtime) |

## Data Shapes

States covered: NoBlock, ReadyToStart, ActiveFran, AmrapRunning.
