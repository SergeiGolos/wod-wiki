# Tracker — Web

The desktop-viewport layout for the workout tracker; renders `TimerScreen` and `VisualStatePanel` side-by-side with a real `ScriptRuntime` backed by a JIT-compiled workout script — nothing is mocked at the component level.

## Depends On

| System / Token | Usage |
|----------------|-------|
| `TimerScreen` | Timer controls (start, pause, stop, next) |
| `VisualStatePanel` | Runtime stack visual state |
| `ScriptRuntime` / `JitCompiler` | Real runtime; no component-level mocks |

## Notifies

| Callback / Event | Trigger | Payload |
|-----------------|---------|---------|
| Start / Pause / Stop / Next | Control buttons | none (dispatched to runtime) |

## Data Shapes

States covered: NoBlock, ReadyToStart, ActiveFran, AmrapRunning.
