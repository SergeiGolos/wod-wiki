## Basic Syntax Guide

### Timers

Primary reason for WOD Wiki, timers to track your workouts, that requires [[timers]].  Lets put 30 seconds on the clock:
<DemoWodContainer>
<pre>:30</pre>
</DemoWodContainer>

Some key features:
 - Time format is `days:hours:minutes:seconds` with a minimal of `:seconds`
 - Timers by default count down

### Stopwatch

Countdowns are great, but workouts often call for a stopwatch instead, which starts from 0 and counts up indefinitely.

<DemoWodContainer>
<pre>^</pre>
</DemoWodContainer>

### Bounded Stopwatch

Combining the a timer with a stopwatch creates a countup timer.

<DemoWodContainer>
<pre>^:30</pre>
</DemoWodContainer>


## Timer Language Features

- **Headings**: Use `#`, `##`, or `###` for different heading levels

- **Rounds**: Use (n) where n is the number of rounds
- **Durations**: Use standard time format `mm:ss` or `hh:mm:ss`
- **Weights**: Use `@` followed by weight and optional unit (kg/lb)
- **Effort **: Simply describe what you are doing in text (e.g., "Push ups", "Deadlifts", "Sun Saluations", etc.)

### Tips for Writing Workouts

1. Start with a clear heading describing the workout type
2. Specify rounds or time domains upfront
3. List movements with specific rep schemes
4. Include weight/loading information where relevant
5. Add notes about movement standards or scaling options

### Example Workouts

Here's a simple EMOM (Every Minute On the Minute) workout:

<WodContainer>
<pre># EMOM 10:00
10 pushups
15 air squats</pre>
</WodContainer>

A more complex workout with rounds and weights:

<WodContainer>
<pre># For Time (3)
15 deadlifts @225lb
12 burpees
9 pull-ups</pre>
</WodContainer>



The parser will automatically recognize and format your workout appropriately based on these syntax rules.
