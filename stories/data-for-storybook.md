## Workout & Effort Data

### `IEffort` — Effort registry entries

**Type:** `src/core/contracts/IEffort.ts` (or `@/effort-registry`)  
**Used by:** `EffortCard`, `EffortDetailPage`, `EffortsCatalogPage`

| Scenario | Origin | Discipline | Aliases | Description |
|----------|--------|------------|---------|-------------|
| **Bundled benchmark** | `bundled` | `crossfit` | `["Fran"]` | Read-only, shows origin badge |
| **User custom** | `user` | `strength` | `["Back Squat", "Squat"]` | Editable, shows derivation parent |
| **Synthetic unresolved** | `synthetic` | `crossfit` | `["Unknown AMRAP"]` | Estimated placeholder, italicized |
| **High intensity** | `user` | `crossfit` | `["DT","D.T.","Dt Complex"]` | Many aliases, compact chip layout |
| **Minimal** | `bundled` | `crossfit` | `[]` | No discipline, no aliases, bare card |
| **With image** | `user` | `gymnastics` | `["Muscle-up"]` | `image` field populated |

```ts
// Example — bundled benchmark
const bundledEffort: IEffort = {
  id: 'fran',
  name: 'Fran',
  origin: 'bundled',
  discipline: 'crossfit',
  aliases: ['Fran'],
  description: '21-15-9 Thrusters & Pull-ups',
  image: '/images/fran.jpg',
};
```

## Metric & Runtime Data

### `IMetric` — Single measurement fragment

**Type:** `src/core/models/Metric.ts` → `IMetric`  
**Used by:** `MetricPill`, `MetricVisualizer`, `MetricSourceRow`, `MetricTrackerCard`, `ReviewGrid`, `StatementDisplay`

| Type | Value | Origin | Use case |
|------|-------|--------|----------|
| `rep` | `21` | `parser` | Standard repetition count |
| `resistance` | `"95lb"` | `parser` | Load prescription |
| `action` | `"Thrusters"` | `parser` | Movement name |
| `rounds` | `3` | `runtime` | Completed rounds in AMRAP |
| `time` | `1200000` (20 min) | `runtime` | Elapsed time in ms |
| `distance` | `400` | `parser` | Meters |
| `calories` | `15` | `user` | User-entered calories |
| `CurrentRound` | `{round:2,total:5}` | `runtime` | Complex object value |
| `Text` | `"Rest 2:00"` | `compiler` | Instruction text |
| `pace` | `"4:30/mi"` | `analyzed` | Derived metric |

```ts
// Helper used across stories
function m(type: string, value: unknown, origin: IMetric['origin'] = 'parser'): IMetric {
  return { type, value, origin } as IMetric;
}
```

### `ICodeStatement` — Parsed statement node

**Type:** `src/runtime/contracts/ICodeStatement.ts`  
**Used by:** `StatementDisplay`, `StatementList`, `MetricSourceRow`

Live cases:
- **Simple statement:** `5 Pull-ups` → 2 metrics (`rep`, `action`)
- **Compound statement:** `Back Squat 5×225lb` → 3 metrics (`action`, `rep`, `resistance`)
- **Rest statement:** `Rest 2:00` → `time` metric, `isRest: true`
- **Header statement:** `(21-15-9)` → 0 metrics, `isHeader: true`

### `AnalyticsSegment` / `AnalyticsGroup` — Review grid data

**Type:** `src/core/models/AnalyticsModels.ts`  
**Used by:** `ReviewGrid`, `ResultsView`, `AnalyticsScorecard`

Generated by driving `ScriptRuntime` to completion. Each story should use real workout scripts:

| Workout | Segments | Groups | Notable traits |
|---------|----------|--------|----------------|
| **Fran** | 6 | 3 | 21-15-9 rep scheme |
| **Cindy** | varies | 3 | AMRAP 20 min, rounds metric |
| **Annie** | 10 | 2 | 50-40-30-20-10 descending |
| **Barbara** | 20 | 5 | 5 rounds + timed rest |
| **Murph** | 4 | 4 | Mile run + 20 rounds + mile run |
| **Grace** | 1 | 1 | 30 Clean & Jerks for time |
| **Karen** | 1 | 1 | 150 Wall Balls for time |

```ts
// Runtime helper pattern (from ReviewGrid.stories.tsx)
function runToCompletion(
  scriptText: string,
  stepMs = 30_000,
  maxSteps = 50,
```

## Where to add new fixtures

1. **If it is reused by ≥ 2 story files** → add to `stories/_shared/fixtures.ts`
2. **If it is domain-specific but complex** → add to this file under the right heading, then import into a story-specific helper file
3. **If it is single-story-only and < 10 lines** → keep inline in the story file

## Type-to-Story Linking Convention

Every story file JSDoc header should include:

```ts
/**
 * Catalog / {Tier} / {ComponentName}
 *
 * Renders: {@link import('@/path/to/Component').ComponentName}
 * Types:    {@link import('@/path/to/types').IMetric}
 * Data:     See {@link ../data-for-storybook.md#metric--runtime-data}
 *
 * Stories:
 *  1. ScenarioName — why this scenario matters
 *  2. ...
 */
```

This makes it trivial to navigate from a story to its backing types and data.
